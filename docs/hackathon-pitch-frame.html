01
The Problem · 痛点大算力机器，普通人指挥不动
  • DGX Spark 算力强，但日常只能 SSH 命令行操作
  • 团队 / 非工程同学 无法安全便捷地驱动它出图、出视频、跑模型
  • 要么给谁都放开权限（危险），要么谁都碰不了（低效）

02
The Idea · 一句话解法把飞书变成这台机器的安全遥控器
One Sentence

发一句话给飞书 bot，系统自动完成 意图识别 → 危险拦截 → 分级审批 → 执行 → 记账 的全链路。

  • 人人可用　在飞书里说人话，不碰命令行
  • 危险必拦　灾难级请求（rm -rf /、外发凭证）直接拒绝
  • 敏感必审　部署等动作先发审批卡，点「批准」才跑

The System · 系统全景三个模块，一台机器
Architecture · 飞书为入口的一体化智能体
04
★ Highlight · 核心亮点猫管家 = 一个 bot，安全管多个项目
插件化多项目管理

插件化架构：未来多接一个项目 = 多写一个插件文件。一个飞书总控 bot，就能安全地统管这台 Spark 上的所有项目。

One Butler · Many Projects
  • 插件自动发现　plugins/ 里放一个文件即接入，can_handle 打分路由
  • 只读与动作分离　运维查询走只读插件，部署 / 生成一律回落安全门
  • 统一管控面　多个项目共用同一套审批门 / 台账 / 记忆，不各搞一套
  • 可横向扩展　新项目上机，写个插件就纳入猫管家统一指挥

07
Platform Fit · 吃满平台为什么非 DGX Spark 不可
128GB Unified Memory

正是 GB10 的大统一内存，让 22B 视频模型 + Gemma-3-12B 编码器 + Flux2 能同机常驻、不换进换出。

  • 量化落地　LTX-2.3 用 fp8 权重 + distilled LoRA，单卡跑全链路
  • 算力分配　扩散模型独占 GB10；对话 / 规划走云端 StepFun，不抢显存
  • ARM 原生　纯 Python，无 Node、无桌面依赖，最贴无头 ARM 服务器
