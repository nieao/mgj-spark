# 技术栈说明

> 列明项目所使用的 NVIDIA SDK / 平台能力、NVIDIA 及开源大模型、以及 StepFun 阶跃星辰大模型。

## 一、NVIDIA 平台 / SDK / 加速能力

| 项 | 使用说明 |
|---|---|
| **DGX Spark（GB10 Grace Blackwell）** | 硬件平台，aarch64 / 128GB 统一内存 / Ubuntu 24.04；2 台组网 |
| **CUDA on ARM** | GB10 单卡驱动扩散模型推理（Flux2 / LTX / Qwen-Image-Edit） |
| **NVFP4 精度格式** | GB10 Blackwell 原生支持的 FP4 硬件加速，用于 StepFun 与 ComfyUI 的低精度高效推理 |
| **NVIDIA Container Toolkit / Docker GPU** | aeon-spark ComfyUI 容器化，换取 ARM 依赖确定性 |
| **NVML（`nvidia-smi`）** | 观测层实时采集 GPU 名称/温度/利用率/显存，接入黑金观测看板 |
| **PyTorch（CUDA build）** | ComfyUI 推理引擎底座，跑在 GB10 |

## 二、在 NVIDIA 平台上运行的模型（视觉 / 视频 / 编码）

| 模型 | 角色 |
|---|---|
| **Flux2（含 Klein 档）** | 文生图主力，稳定出图 |
| **Qwen-Image-Edit** | 多参考图编辑 |
| **LTX-2.3 22B（fp8 + distilled LoRA）** | 带音频文生视频核心，t2v + i2v 一致性流 |
| **Gemma-3-12B** | LTX 文本编码器 |

> 说明：以上开源模型均运行于 NVIDIA DGX Spark 平台，充分利用 GB10 的 CUDA / NVFP4 / 统一内存能力；具备 NVIDIA 多模态模型的接入位，可作为内容理解/提示词生成的备选。

## 三、StepFun 阶跃星辰大模型

| 项 | 使用说明 |
|---|---|
| **Step-3.7-Flash** | 项目默认对话大模型：意图定档 / 多步规划（iterate）/ 兜底问答（general_qa） |
| **本地部署** | 一台 DGX Spark 上以 NVFP4 / Q3_K_M（94GB）量化本地运行，OpenAI 兼容 HTTP，纯本地多模态推理 |
| **云端 API 兜底** | 本地失败或资源受限时自动切阶跃公网 API，保证演示不中断 |

## 四、智能体 / 后端 / 运维框架

| 层 | 使用的 SDK / 工具 |
|---|---|
| **智能体中枢** | Python 3 · 自研 `bridge.py` 编排 + `plugins/` 插件化 + `core/` 安全内核 |
| **飞书通道** | `lark-oapi`（WebSocket 长连接接收）+ tenant_access_token REST（发送）；可切 node lark-cli |
| **生成引擎** | ComfyUI（官方 API / `/history` 黄金工作流）+ `ffmpeg`（分段视频合并） |
| **AIGC 工作台** | FastAPI |
| **稳定传输** | paramiko（scp / 远程执行） |
| **运维 / 网络** | systemd（三层守护自愈）· Cloudflare Tunnel（cloudflared）· Caddy（Basic Auth） |
